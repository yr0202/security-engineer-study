# RSA와 모듈러 연산

> 📅 2026.04.08 | 📂 정보보안일반

---

## 📖 개념 요약
RSA는 공개키 암호화 방식으로, 큰 수의 소인수분해가 어렵다는 수학적 특성을 이용한다.
모듈러 연산(나머지 연산)이 핵심 수학 원리로 사용된다.

---


## 🔑 모듈러 연산 기초

### 모듈러 연산이란?
- `a mod n` = a를 n으로 나눈 **나머지**
- 예: `17 mod 5 = 2`

### 기본 성질
```
(a + b) mod n = ((a mod n) + (b mod n)) mod n
(a × b) mod n = ((a mod n) × (b mod n)) mod n
```

---

## 🔑 RSA 키 생성 과정 (암기 필수)

### 1단계 - 두 소수 선택 (문제 제시)
```
p = 5, q = 11  (실제로는 매우 큰 소수 사용)
```

### 2단계 - n 계산 (공개키의 일부)
```
n = p × q = 5 × 11 = 55
```

### 3단계 - 오일러 함수 φ(n) 계산
```
φ(n) = (p-1) × (q-1) = 4 × 10 = 40
```

### 4단계 - 공개키 e 선택 (문제 제시)
```
조건: 1 < e < φ(n), e와 φ(n) 은 서로소
e = 7  (7과 40은 서로소)
```

### 5단계 - 개인키 d 계산
```
조건: (e × d) mod φ(n) = 1
(7 × d) mod 40 = 1
d = 23  → 7 × 23 = 161, 161 mod 40 = 1 ✅
```

### 결과
```
공개키: (e, n) = (7, 55)
개인키: (d, n) = (23, 55)
```


---

## 🔑 암호화 / 복호화

### 용어 정리
- **M = Message = 평문** (원본 메시지, 암호화 전)
- **C = Ciphertext = 암호문** (암호화된 메시지)

### 암호화 (공개키 사용)
```
C = M^e mod n
M = 2 (제시)
C = 2^7 mod 55 = 128 mod 55 = 18
```

### 복호화 (개인키 사용)
```
M = C^d mod n
M = 18^23 mod 55 = 2 ✅
```

---

## ⚠️ 시험 포인트
- RSA는 **비대칭키(공개키)** 암호화 방식
- 암호화 → 공개키(e, n) 사용
- 복호화 → 개인키(d, n) 사용
- `φ(n) = (p-1)(q-1)` 공식 암기 필수
- `(e × d) mod φ(n) = 1` 관계 암기
- 소인수분해의 어려움 = RSA 안전성의 근거

---

## 🔑 키 사용 방향 (중요!)

### 일반적인 경우 (기밀성)
| 목적 | 사용 키 |
|------|--------|
| 암호화 | 공개키 (e, n) |
| 복호화 | 개인키 (d, n) |

→ 상대방한테 메시지를 **비밀로 보낼 때** 사용
→ 받는 사람의 공개키로 암호화 → 받는 사람만 개인키로 복호화 가능

### 반대의 경우 (전자서명)
| 목적 | 사용 키 |
|------|--------|
| 서명 (암호화) | 개인키 (d, n) |
| 검증 (복호화) | 공개키 (e, n) |

→ **"내가 보낸 게 맞다"** 를 증명할 때 사용
→ 내 개인키로 서명 → 누구든 내 공개키로 검증 가능

> RSA는 수학적으로 공개키 ↔ 개인키 양방향 모두 동작
> 어떤 키로 암호화하느냐에 따라 **용도**가 달라지는 것!

## ❓ 헷갈렸던 것
- e와 d 역할 혼동 주의 → **e(encrypt/공개), d(decrypt/개인)**
- n은 공개키와 개인키 **둘 다** 포함됨
- 암호화/복호화 방향이 고정된 게 아니라 **목적에 따라 달라짐**
- 기밀성 목적 → 공개키로 암호화 / 인증·서명 목적 → 개인키로 암호화
