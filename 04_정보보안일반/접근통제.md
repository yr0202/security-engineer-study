# 접근통제 (Access Control)

## 1. 접근통제 기본 개념

### 정의
- 주체(Subject)가 객체(Object)에 접근할 때, 허가된 방식으로만 접근하도록 제한하는 보안 메커니즘

### 구성 요소
| 구성 요소 | 설명 | 예시 |
|-----------|------|------|
| **주체 (Subject)** | 접근을 요청하는 능동적 존재 | 사용자, 프로세스, 프로그램 |
| **객체 (Object)** | 접근 대상이 되는 수동적 존재 | 파일, DB, 네트워크, 장치 |
| **접근 (Access)** | 주체가 객체에 행하는 행위 | 읽기, 쓰기, 실행, 삭제 |

### 접근통제 3단계 (AAA)
1. **식별 (Identification)** : 주체가 자신을 시스템에 밝히는 행위 (ID 입력)
2. **인증 (Authentication)** : 주체가 주장한 신원을 검증하는 행위 (PW 확인)
3. **인가 (Authorization)** : 인증된 주체에게 접근 권한을 부여하는 행위

> 식별 → 인증 → 인가 순서로 진행

## 2. 접근통제 정책 (모델)

### 2-1. 강제적 접근통제 (MAC, Mandatory Access Control)
- **개념** : 시스템이 보안 레이블(등급)을 기반으로 접근을 강제 통제
- **결정 주체** : 시스템 (관리자가 정책 설정, 사용자 변경 불가)
- **기반** : 보안 등급(Security Label) — 주체와 객체 모두에 부여
- **특징**
  - 군사·정부 기관에서 주로 사용
  - 높은 보안성, 낮은 유연성
  - 구현 복잡, 비용 높음
- **대표 모델** : Bell-LaPadula (기밀성), Biba (무결성)

### 2-2. 임의적 접근통제 (DAC, Discretionary Access Control)
- **개념** : 객체의 소유자가 자신의 재량으로 접근 권한을 설정
- **결정 주체** : 객체 소유자 (Owner)
- **기반** : 신원(Identity) — ACL(접근제어 목록) 사용
- **특징**
  - 유닉스/윈도우 파일시스템에서 사용
  - 유연하지만 트로이 목마 공격에 취약
  - 권한 분산으로 관리 어려움
- **구현** : ACL(Access Control List), 허가 비트(rwx)

### 2-3. 역할기반 접근통제 (RBAC, Role-Based Access Control)
- **개념** : 사용자의 역할(Role)에 따라 접근 권한을 부여
- **결정 주체** : 관리자 (역할 정의), 사용자는 역할에 배정됨
- **특징**
  - 최소 권한 원칙 적용 용이
  - 기업 환경에서 가장 널리 사용
  - 역할 계층구조 지원
  - 상호 배타적 역할(SoD) 지원
- **구성** : 사용자 ↔ 역할 ↔ 권한

### 2-4. 속성기반 접근통제 (ABAC, Attribute-Based Access Control)
- **개념** : 사용자·자원·환경의 속성(Attribute)을 조합하여 접근 결정
- **특징**
  - 가장 유연하고 세밀한 통제 가능
  - 정책 표현이 복잡함
  - XACML 등의 표준 사용

### 비교 요약

| 구분 | MAC | DAC | RBAC |
|------|-----|-----|------|
| 권한 결정 | 시스템/관리자 | 소유자 | 관리자(역할 기반) |
| 기반 | 보안 등급(레이블) | 신원(ID) | 역할(Role) |
| 유연성 | 낮음 | 높음 | 중간 |
| 보안성 | 높음 | 낮음 | 중간 |
| 사용 환경 | 군사/정부 | 일반 OS | 기업 환경 |

## 3. 인증 방식

### 인증 요소 (Factor)
| 요소 | 설명 | 예시 |
|------|------|------|
| **지식 기반 (Something you know)** | 알고 있는 것 | 비밀번호, PIN |
| **소유 기반 (Something you have)** | 가진 것 | OTP 토큰, 스마트카드 |
| **존재 기반 (Something you are)** | 생체 특성 | 지문, 홍채, 얼굴 |
| **행위 기반 (Something you do)** | 행동 패턴 | 서명, 타이핑 패턴 |
| **위치 기반 (Somewhere you are)** | 위치 정보 | GPS, IP 주소 |

### 다중 인증 (MFA)
- **2FA** : 두 가지 인증 요소 사용
- **MFA** : 두 가지 이상 인증 요소 사용
- 요소를 결합할수록 보안 강도 증가

### 생체 인증 오류율
- **FRR (False Rejection Rate, 오거부율)** : 정당한 사용자를 거부하는 비율 → 편의성과 반비례
- **FAR (False Acceptance Rate, 오인증율)** : 불법 사용자를 허용하는 비율 → 보안성과 반비례
- **CER (Crossover Error Rate, 교차오류율)** : FRR = FAR 이 되는 지점 → 낮을수록 우수한 시스템

```
FRR ↑ = 보안성 ↑, 편의성 ↓
FAR ↑ = 보안성 ↓, 편의성 ↑
CER : 두 오류율이 같아지는 지점 (성능 기준점)
```

## 4. 접근통제 원칙

### 최소 권한 원칙 (Least Privilege)
- 작업 수행에 필요한 최소한의 권한만 부여
- 권한 남용, 실수, 악성코드 확산 방지

### 직무 분리 (Separation of Duties, SoD)
- 중요한 업무를 여러 사람이 나누어 처리
- 내부자 사기 및 오류 방지
- 예: 결재자와 지불자 분리

### 알 필요성 원칙 (Need to Know)
- 업무 수행에 필요한 정보에만 접근 허용
- 기밀성 강화

### 직무 순환 (Job Rotation)
- 담당자를 주기적으로 교체
- 장기 공모 방지, 부정행위 발견 가능성 증가

## 5. 접근통제 구현 기술

### ACL (Access Control List)
- 객체에 연결된 접근 권한 목록
- 각 주체별 허용/거부 권한 명시

### RBAC 구성 요소
- Users → Roles → Permissions
- Role 계층구조: 상위 역할이 하위 역할의 권한 포함

### 싱글사인온 (SSO, Single Sign-On)
- 한 번 인증으로 여러 시스템 접근
- 장점: 편의성 증가, 관리 단순화
- 단점: 단일 실패 지점(SPOF)

### LDAP (Lightweight Directory Access Protocol)
- 디렉터리 서비스 접근 프로토콜
- 사용자 인증·인가 정보 중앙 관리

## 6. 시험 자주 출제 포인트 정리

| 구분 | 핵심 키워드 |
|------|------------|
| MAC | 보안 등급, 레이블, 강제, 군사 환경 |
| DAC | 소유자, 재량, ACL, 트로이 목마 취약 |
| RBAC | 역할, 최소 권한, 기업 환경, 직무 분리 |
| CER | FRR=FAR 교차점, 낮을수록 우수 |
| 최소 권한 | 필요 최소한의 권한만 부여 |
| 직무 분리 | 중요 업무 분담, 내부 부정 방지 |
